# 工作经历增强功能测试指南

## 功能概述

为工作经历模块添加了以下增强功能：

### 新增字段（全部选填）

1. **订单号** - 自动生成，可手动修改
   - 格式：`WO-YYYYMMDD-XXXX`
   - 示例：`WO-20230615-8234`

2. **服务区域** - 北京市16个区的下拉选择
   - 东城区、西城区、朝阳区、海淀区、丰台区、石景山区
   - 通州区、顺义区、昌平区、大兴区、房山区、门头沟区
   - 平谷区、怀柔区、密云区、延庆区

3. **客户姓名** - 文本输入

4. **客户评价** - 多行文本输入

5. **工作照片** - 支持多图上传（最多9张）
   - 支持拖拽排序
   - 自动压缩
   - 支持预览和删除

## 测试步骤

### 1. 创建新简历时测试

1. 登录系统，进入"创建简历"页面
2. 填写基本信息（姓名、年龄、电话等必填项）
3. 滚动到"工作经验"部分
4. 填写第一条工作经历：
   - 选择开始日期和结束日期
   - 输入工作描述
   - 点击"订单号"字段的"生成"按钮，验证订单号自动生成
   - 选择服务区域（如：朝阳区）
   - 输入客户姓名
   - 输入客户评价
   - 上传1-3张工作照片
5. 点击"添加工作经历"按钮，添加第二条工作经历
6. 保存简历
7. 验证保存成功

### 2. 编辑现有简历时测试

1. 进入"简历列表"页面
2. 选择一条简历，点击"编辑"
3. 滚动到"工作经验"部分
4. 编辑现有工作经历：
   - 修改订单号
   - 更改服务区域
   - 更新客户姓名和评价
   - 添加或删除工作照片
5. 保存修改
6. 刷新页面，验证修改已保存

### 3. 工作照片功能测试

1. 在工作经历中点击"工作照片"上传区域
2. 测试多选上传（选择2-3张图片）
3. 验证图片自动压缩
4. 测试拖拽排序功能
5. 测试图片预览功能
6. 测试删除图片功能
7. 测试最大数量限制（9张）

### 4. 订单号生成测试

1. 创建新工作经历
2. 不填写订单号，直接保存（验证选填）
3. 点击"生成"按钮，验证订单号格式正确
4. 手动修改订单号，验证可以修改
5. 修改开始日期后，重新生成订单号，验证日期部分更新

### 5. 区域选择测试

1. 点击"服务区域"下拉框
2. 验证显示16个北京市区
3. 选择不同区域，验证可以正常选择
4. 清空选择，验证可以留空（选填）

### 6. 数据持久化测试

1. 创建包含所有新字段的工作经历
2. 保存简历
3. 退出登录
4. 重新登录
5. 查看简历详情
6. 验证所有新字段数据正确显示

## 预期结果

### 成功标准

- ✅ 所有新字段都是选填的，不影响现有简历创建流程
- ✅ 订单号可以自动生成，格式正确
- ✅ 订单号可以手动修改
- ✅ 服务区域下拉框显示16个北京市区
- ✅ 工作照片支持多图上传（最多9张）
- ✅ 工作照片支持拖拽排序
- ✅ 工作照片自动压缩
- ✅ 所有数据正确保存和加载
- ✅ 前后端编译无错误

## 注意事项

1. 所有新增字段都是**选填**的，不会影响现有功能
2. 工作照片上传会自动压缩，减少存储空间
3. 订单号格式固定为 `WO-YYYYMMDD-XXXX`
4. 服务区域目前仅支持北京市16个区

## 已知限制

1. 工作照片最多支持9张
2. 服务区域仅限北京市
3. 订单号格式固定，不可自定义

## 后续优化建议

1. 支持更多城市的区域选择
2. 订单号格式可配置
3. 工作照片数量限制可配置
4. 添加客户评价星级评分
5. 支持工作照片批量上传
