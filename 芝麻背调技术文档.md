# 芝麻背调技术文档

## 一、概述

**芝麻背调**是 TuringData（图腾数据）提供的候选人背景调查服务，用于核查候选人的学历、工作履历、身份信息等。

- **服务商**: 图腾数据（TuringData）
- **官网**: https://www.turingdata.cn
- **应用场景**: 员工入职前的背景调查

---

## 二、环境配置

| 环境 | 服务器地址 |
|------|------------|
| **测试环境** | `https://test-api.turingdata.cn` |
| **生产环境** | `https://api.turingdata.cn` |

**注意**：
- 需要提前配置 **IP 白名单**
- 联系芝麻背调获取 `appId` 和 `appSecret`

---

## 三、接口规范

### 3.1 通用说明

- **协议**: HTTP
- **请求格式**: JSON（`Content-Type: application/json`）
- **响应格式**: JSON
- **鉴权方式**: 请求头 Header 鉴权

### 3.2 通用 Header 参数

| 参数名 | 必选 | 类型 | 说明 |
|--------|------|------|------|
| `auth-key` | 是 | string | 芝麻背调提供的 appId |
| `auth-timestamp` | 是 | string | 时间戳 `System.currentTimeMillis()` |
| `auth-sign` | 是 | string | 签名（见下方加密方法） |
| `sub-user-id` | 否 | string | 子账号 ID（子账号调用时必传） |
| `proxy-sub-user-id` | 否 | string | 代理子账号 ID（代理子账号调用时必传） |

### 3.3 响应格式

**成功响应**:
```json
{
  "code": 200,
  "data": { }
}
```

**失败响应**:
```json
{
  "code": 500,
  "errorCode": "",
  "errorMsg": "错误信息"
}
```

## 四、签名算法

### 4.1 签名公式

```
auth-sign = SHA1(appId + timestamp + secret)
```

### 4.2 Java 实现

```java
import java.security.MessageDigest;

public class SHAUtils {
    private static final String CHARSET = "utf-8";
    private static final String SHA1 = "SHA1";
    
    public static String encode(String plainText) {
        MessageDigest messageDigest = null;
        try {
            messageDigest = MessageDigest.getInstance(SHA1);
            messageDigest.update(plainText.getBytes(CHARSET));
        } catch (Exception e) {
            return null;
        }
        byte[] bytes = messageDigest.digest();
        return HexUtils.byte2Hex(bytes);
    }
}
```

### 4.3 Node.js/TypeScript 实现

```typescript
import * as crypto from 'crypto';

function generateSignature(appId: string, timestamp: string, secret: string): string {
  const signString = appId + timestamp + secret;
  return crypto.createHash('sha1').update(signString).digest('hex').toUpperCase();
}
```

### 4.4 PHP 实现

```php
$timestamp = microtime();
$sign = sha1($appId . $timestamp . $appSecret);
```

## 八、代码示例

### 8.1 Node.js/TypeScript

```typescript
import axios from 'axios';
import * as crypto from 'crypto';

class ZmdbClient {
  private appId: string;
  private appSecret: string;
  private baseUrl: string;

  constructor(appId: string, appSecret: string, isTest = true) {
    this.appId = appId;
    this.appSecret = appSecret;
    this.baseUrl = isTest 
      ? 'https://test-api.turingdata.cn' 
      : 'https://api.turingdata.cn';
  }

  private generateSign(timestamp: string): string {
    const signString = this.appId + timestamp + this.appSecret;
    return crypto.createHash('sha1').update(signString).digest('hex').toUpperCase();
  }

  private async request(endpoint: string, data: any) {
    const timestamp = Date.now().toString();
    const sign = this.generateSign(timestamp);

    const response = await axios.post(
      `${this.baseUrl}${endpoint}`,
      data,
      {
        headers: {
          'Content-Type': 'application/json',
          'auth-key': this.appId,
          'auth-timestamp': timestamp,
          'auth-sign': sign,
        },
      }
    );
    return response.data;
  }

  // 发起背调
  async queryReport(params: {
    tplId: string;
    platformUserId: string;
    authType: string;
    name: string;
    mobile: string;
    idNo?: string;
  }) {
    return this.request('/platform_report_api/query_report', {
      chnlId: '2',
      terType: '7',
      ...params,
    });
  }

  // 取消背调
  async cancelQuery(reportId: string) {
    return this.request('/platform_report_api/cancel_query', { reportId });
  }
}

// 使用
const client = new ZmdbClient('your-appId', 'your-appSecret', true);
const result = await client.queryReport({
  tplId: '202111150001214809',
  platformUserId: '202106210001033825',
  authType: '3',
  name: '张三',
  mobile: '18827529397',
});
```

## 九、集成建议

### 9.1 在你的 CRM 中集成

1. **创建背调模块**: `backend/src/modules/zmdb/`
2. **配置环境变量**:
   ```
   ZMDB_APP_ID=your_app_id
   ZMDB_APP_SECRET=your_app_secret
   ZMDB_IS_TEST=true
   ```
3. **业务流程**:
   - 员工发起背调 → 调用 `query_report`
   - 获取 `reportId` → 保存到数据库
   - 配置回调地址 → 接收状态通知
   - 报告完成 → 下载 PDF / 展示结果

### 9.2 注意事项

- 需要在芝麻背调控制台配置**IP 白名单**
- 需要配置**回调通知地址**接收报告状态变更
- 批量背调适合大规模招聘场景
- 报告生成需要时间，建议使用异步回调机制
